# Marmotte — AI Assistant Guide

Marmotte is an **extensible development toolkit for TypeScript projects**. It ships a collection of composable Vite plugins that handle common concerns: library bundling, Vue/Vuetify UI setup, VitePress documentation, TypeDoc API generation, and Vitest utilities.

## Repository layout

```
src/
  vite/                  # Vite plugins (the core of the library)
    lib.ts               # Lib() — build a TS library
    ui.ts                # UILib() / UIApp() — Vue + Vuetify
    docs/                # Docs() — VitePress dev server + build
    dts.ts               # thin wrapper around vite-plugin-dts
    externals.ts         # re-exports rollup-plugin-node-externals
    base-config.ts       # BaseConfig() / BaseBundle() shared baseline
    package-meta/        # PackageMeta() plugin + client helper
    lib/context.ts       # VitePluginContext base class (path resolution)
  vitepress/
    typedoc.ts           # TypeDocPlugin() — generate API docs
    sidebar.ts           # re-exports vitepress-sidebar
  vitest/
    index.ts             # withTmpDir(), createTmpDir()
  utils/
    fs.ts                # walk(), resolveEntries(), fileExists(), …
    codegen/             # header comments, file writing helpers
    package-json.ts      # Zod schema for package.json
docs/                    # VitePress site (partially auto-generated)
tests/                   # Vitest tests
vite.config.ts           # builds the library itself
```

## Key patterns and conventions

### Plugin composition

Plugins are ordinary arrays of Vite `Plugin` objects. Higher-level plugins (`Lib`, `UILib`) delegate to lower-level ones and accept their options as pass-through:

```ts
// Lib() = BaseBundle() + dts() + nodeExternals() + Docs() + LibConfig()
export function Lib(options: LibPluginOptions = {}) { … }
```

### VitePluginContext — path resolution

All plugins that need filesystem paths use `VitePluginContext` (or the pre-configured `DefaultVitePluginContext`). It provides typed, root-relative path resolution:

```ts
const ctx = new DefaultVitePluginContext({ root: cfg.root });
ctx.resolve("sourceDir", "index.ts"); // → /abs/path/src/index.ts
```

`DefaultVitePluginContext` maps `sourceDir → src/`. Use `contextFactory()` to create contexts with custom path names.

### Auto-managed vs default files

- **Auto-managed** (`writeAutoGeneratedFile`): always overwritten by the tool; humans should not edit them; a `.bak` is created if a manual edit is detected.
- **Default** (`writeDefaultFile`): written once; users are free to edit; will not be overwritten.

### Entry point discovery

`resolveEntries(dir, filter)` walks a directory and returns a Rollup entry map. `filter` is a `RegExp` or `(path: string) => boolean`.

### Comment type detection

`resolveCommentType(filePath)` auto-detects `//`, `#`, or `<!-- -->` based on file extension. New types can be registered with `registerCommentTypeResolver()`.

## Adding a new entry point to the library

1. Create `src/<module>/index.ts` (or the file itself).
2. Add it to the `entry` object in [vite.config.ts](vite.config.ts).
3. Add the matching `exports` key in [package.json](package.json).
4. Run `npm run build` to verify.

## Adding a new Vite plugin

1. Create `src/vite/<name>.ts` and export a `function MyPlugin(options): Plugin | Plugin[]`.
2. Use `DefaultVitePluginContext` for path resolution if needed.
3. Add to the `entry` map in [vite.config.ts](vite.config.ts) under `vite/<name>`.
4. Add the `exports` entry in [package.json](package.json): `"./vite/<name>": { "types": "...", "import": "..." }`.
5. Write a test in `tests/`.

## Scripts

| Command              | Description                            |
| -------------------- | -------------------------------------- |
| `npm run build`      | Build the library to `dist/`           |
| `npm run dev`        | Build in watch mode                    |
| `npm run docs`       | Start VitePress dev server for `docs/` |
| `npm run build:docs` | Build the VitePress site               |
| `npm run test`       | Run Vitest tests                       |
| `npm run lint`       | Lint with oxlint                       |
| `npm run fmt`        | Format with oxfmt                      |

## Testing

Tests live in `tests/`. Use `withTmpDir()` from `marmotte/vitest` to get an auto-cleaned temporary directory in hooks:

```ts
import { withTmpDir } from "marmotte/vitest";
const { dir } = withTmpDir();
```


## Toolchain

- **Bundler**: Vite (ESM-only output, no minification, sourcemaps on, modules preserved)
- **Types**: `vite-plugin-dts` — errors on type failures
- **Linter**: oxlint
- **Formatter**: oxfmt
- **Docs**: VitePress + TypeDoc (auto-generated API reference)
- **Package schema**: Zod (`ZPackageJson` in `src/utils/package-json.ts`)
